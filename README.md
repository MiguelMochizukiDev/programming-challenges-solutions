# Programming Challenges Solutions

Solutions for problems from Steven S. Skiena and Miguel A. Revilla's *Programming Challenges*, implemented in C, C++, Java, and Python.

## Languages Used

- C (GCC, C11)
- C++ (G++, C++17)
- Java (JDK with `javac`/`java`)
- Python (`python3`)

### Problem Layout Conventions

For each problem folder `chapters/pcXX/pcXXXX/`:

- `c/`: C solution source and generated `bin/` artifacts.
- `cpp/`: C++ solution source and generated `bin/` artifacts.
- `java/`: Java solution source and generated `bin/` classes.
- `python/`: Python solution source and generated `__pycache__/`.
- `sample_data/input.txt`: sample input used by test runner.
- `sample_data/output.txt`: expected output for sample input.

`bin/` and `__pycache__/` are generated by build/test commands.

## Requirements

Make sure these tools are available in your environment:

- `sh`
- `make`
- `gcc`
- `g++`
- `javac` and `java`
- `python3`
- standard Unix tools (`find`, `diff`, `sed`, `mktemp`)

## Build Solutions

From repository root, use either Make or scripts directly.

### Using Make

```sh
make -C scripts build -- [--chapter pcXX] [--num pcXXXX] [--lang c|cpp|java|python|all]
```

Examples:

```sh
# Build everything in all languages
make -C scripts build

# Build all problems in chapter pc01 for C++
make -C scripts build -- --chapter pc01 --lang cpp

# Build only problem pc0102 in Java
make -C scripts build -- --num pc0102 --lang java
```

### Using Script Directly

```sh
sh scripts/build.sh [--chapter pcXX] [--num pcXXXX] [--lang c|cpp|java|python|all]
```

## Run Tests

Tests are sample-based checks:

1. Build selected solutions (if needed).
2. Execute each solution with `sample_data/input.txt`.
3. Compare produced output against `sample_data/output.txt`.

Run tests with:

```sh
sh scripts/run_test.sh [--chapter pcXX] [--num pcXXXX] [--lang c|cpp|java|python|all]
```

Examples:

```sh
# Test everything
sh scripts/run_test.sh

# Test chapter pc01, Python only
sh scripts/run_test.sh --chapter pc01 --lang python

# Test one problem in all languages
sh scripts/run_test.sh --num pc0101 --lang all
```

The script prints a summary with `pass`, `fail`, and `skip`. It exits with non-zero status when at least one test fails.

## Clean Build Artifacts

### Using Make

```sh
make -C scripts clean -- [--chapter pcXX] [--num pcXXXX] [--lang c|cpp|java|python|all]
```

### Using Script Directly

```sh
sh scripts/clean.sh [--chapter pcXX] [--num pcXXXX] [--lang c|cpp|java|python|all]
```

This removes generated `bin/` folders and Python `__pycache__/` for the selected scope.

## Add, Update, and Maintain Solutions

### Add a New Chapter

1. Create chapter directory: `chapters/pcNN/`.
2. Add one or more problem directories under it (for example `pcNN01`).

### Add a New Problem

1. Create `chapters/pcXX/pcXXXX/`.
2. Add language folders you want to support: `c/`, `cpp/`, `java/`, `python/`.
3. Add sample files:
	- `sample_data/input.txt`
	- `sample_data/output.txt`
4. Add at most one source file per language folder (matching current scripts behavior):
	- C: `pcXXXX.c`
	- C++: `pcXXXX.cpp`
	- Java: `PCXXXX.java`
	- Python: `pcXXXX.py`

### Update an Existing Solution

1. Edit the source file in the corresponding language folder.
2. Rebuild only what changed (optional filters supported).
3. Re-run tests for that problem/chapter/language.

Example workflow:

```sh
sh scripts/build.sh --chapter pc01 --num pc0101 --lang c
sh scripts/run_test.sh --chapter pc01 --num pc0101 --lang c
```

## Script Reference

- `scripts/build.sh`: Compiles/builds selected solutions.
- `scripts/run_test.sh`: Runs sample-based tests and diffs output.
- `scripts/clean.sh`: Removes generated artifacts.
- `scripts/_common.sh`: Shared CLI parsing and chapter/problem iteration helpers.
- `scripts/Makefile`: Make targets wrapping build/clean scripts.

Run help:

```sh
make -C scripts help
sh scripts/run_test.sh --help
```
